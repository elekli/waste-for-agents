Four steps to your first drop

It's open source and runs on your own machine. Python 3.12+, uv, and Claude Code (only for the agent path; the HTTP path needs just curl).

1

Clone & start the server

--unmetered turns off the billing gate — you're on your own machine; you shouldn't be throttled by a free-tier quota meant for the hosted service.

terminal
# landed state goes to ~/.waste-for-agents/
git clone https://github.com/elekli/waste-for-agents.git && cd waste-for-agents
uv run python -m waste_for_agents serve --port 8848 --unmetered &
# health: curl http://127.0.0.1:8848/health  → {"status":"ok"}
2

Issue yourself a key

Writes a key straight to the local DB and prints it. (Sidesteps a chicken-and-egg: the MCP header needs a key, but the self-serve issue_key tool needs you already connected.)

terminal
KEY=$(uv run python -m waste_for_agents issue-key)
echo $KEY   → wfa_…
3

Register the MCP server agent path

!

Order matters. Add the server before you start (or restart) Claude Code. A server registered mid-session does not load its tools into the running session.

terminal
claude mcp add --transport http waste http://127.0.0.1:8848/mcp/ \
  --header "Authorization: Bearer $KEY"
# then start a fresh Claude Code session
4

Subscribe & drain

In your agent: create a watch, then pull changes. Give a homepage URL — discovery finds the feed (it tries <link>, then /feed, /rss…).

in your agent
create_watch(
  source="rss",
  query={"url": "https://news.ycombinator.com/news"},
  key_columns=["id"], ignore_columns=[], interval_s=3600)

list_changes()   # a batch on first wake, then silence until something shifts

One curl into anything

The server polls on its own while it runs. To pipe changes anywhere, poll the HTTP mirror and forward. You hold the cursor between runs; the server holds the diff.

crontab — every 15 min → Slack
# forward new HN changes to a Slack webhook
*/15 * * * *  C=$(cat ~/.wfa_cursor 2>/dev/null); \
  R=$(curl -s "http://127.0.0.1:8848/changes?since=$C" -H "Authorization: Bearer $KEY"); \
  echo $R | jq '.cursor' > ~/.wfa_cursor; \
  echo $R | jq -c '.events[]' | while read e; do curl -s -X POST $SLACK_WEBHOOK -d "$e"; done

A list_changes that returns {"events": [], "cursor": N} is the muted horn — nothing shifted upstream. That's the normal state, not an error.

Field notes

Things that trip people up

SymptomWhat's happeningFix
403 Forbidden on a homepageThe site blocks requests without a browser User-Agent.Add one: headers:{"User-Agent":"Mozilla/5.0 …"}
"no feed found" on a 200 OKNo discoverable <link rel=alternate>, and common paths didn't resolve.Give the feed URL directly (often /feed or /rss).
Agent can't see the toolsThe MCP server was added after the session started.Restart Claude Code after claude mcp add.
Second poll repeats itemslist_changes doesn't track your cursor for you.Pass the returned cursor back as since_cursor.
Want more items than you getNo count knob — item count is whatever the feed publishes (HN ≈ 30, many blogs ≈ 10–15).Use the source's own paginated feed URL, if it has one.
