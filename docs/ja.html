AI エージェントのための
RSS 購読

エージェントが見張りたいフィードのための、頼れる配送係。一度購読すれば、ひとつの静かなチャネルを開いたまま、本当の変化だけを受け取ります。

ローカルで動かす ↓ ソースを見る
 list_changes · 投函
# エージェントが目を覚まし、watch ごとに一度尋ねる
list_changes(watch_id="hn")
# たいていは沈黙 — 上流で何も変わっていない:
{ "events": [], "cursor": 41 }

# そしてある日、ホルンが鳴る:
{ "events": [{ "kind":"added",
    "title":"Show HN: …",
    "content":"<clean markdown>" }],
  "cursor": 42 }
5 分で手に入るもの

あなたが眠る間も動くポーラー

ローカルサーバーが定期的にフィードを巡回し、本当の変化だけを蓄積します。受け渡しは 2 通り:エージェント向けの MCP ツール(Claude Code など)と、素の HTTP エンドポイント — GET /changes — で、curl で cron や Slack の webhook、別のサービスへ流し込めます。

本物の差分 — timestamp のノイズを無視 本文はきれいな Markdown 沈黙こそ no-op
i

前提条件。 Python 3.12+、uv、MCP 対応エージェント — Claude Code・OpenCode・Cursor(エージェント経路のみ必要;HTTP 経路は curl だけで足ります)。

セットアップ · セルフホスト

最初の投函までの 4 ステップ

1

クローンしてサーバーを起動

--unmetered は課金ゲートを無効にします — 自分のマシンで動かしているのですから、ホスト版向けの無料枠で制限される筋合いはありません。

ターミナル
# uv がまだ?  curl -LsSf https://astral.sh/uv/install.sh | sh  — その後シェルを開き直す
# 生成物は ~/.waste-for-agents/ に置かれる
git clone https://github.com/elekli/waste-for-agents.git && cd waste-for-agents
uv run python -m waste_for_agents serve --port 8848 --unmetered &
curl http://127.0.0.1:8848/health   # → {"status":"ok","watches":0}
2

自分用のキーを発行

ローカル DB に直接キーを書き込んで表示します。(エージェント自身でも発行できます — issue_key はキーなしで動く唯一のツールです — が、結局そのキーをクライアント設定に貼って再接続することになります。先に CLI で発行すれば、その往復を省けます。)

ターミナル
KEY=$(uv run python -m waste_for_agents issue-key)
echo $KEY   # → wfa_…
3

MCP サーバーを登録 エージェント経路

!

順序が重要。 エージェントのセッションを起動(または再起動)する 前 にサーバーを登録してください。セッション中に追加したサーバーのツールは読み込まれ ません。また Authorization ヘッダーは必ず付けたままに — ないと issue_key 以外のすべての呼び出しが unauthorized を返します。

ターミナル
claude mcp add --transport http posthorn http://127.0.0.1:8848/mcp/ --header "Authorization: Bearer $KEY"
# その後、新しい Claude Code セッションを開始
opencode.json — プロジェクト直下または (~/.config/opencode/)
{
  "mcp": {
    "posthorn": {
      "type": "remote",
      "url": "http://127.0.0.1:8848/mcp/",
      "headers": { "Authorization": "Bearer wfa_PASTE-YOUR-KEY" },
      "enabled": true
    }
  }
}

JSON は $KEY を展開しません — ステップ 2 のキー(echo $KEY)をそのまま貼り付けて、OpenCode を再起動してください。

.cursor/mcp.json (~/.cursor/mcp.json)
{
  "mcpServers": {
    "posthorn": {
      "url": "http://127.0.0.1:8848/mcp/",
      "headers": { "Authorization": "Bearer wfa_PASTE-YOUR-KEY" }
    }
  }
}

ステップ 2 のキーをそのまま貼り付けて、Cursor Settings → MCP でサーバーを再読み込み(または Cursor を再起動)してください。

4

購読して引き出す

エージェントで:watch を作成し、変化を引き出します。トップページの URL を渡せば、ディスカバリーがフィードを見つけます(<link>、次に /feed、/rss …の順に試します)。

エージェント内で
w = create_watch(
  source="rss",
  query={"url": "https://news.ycombinator.com/news"},
  key_columns=["id"], ignore_columns=[], interval_s=3600)   # → {"watch_id": ...}

list_changes(watch_id=w["watch_id"])   # この watch の変更 — 各 watch は自分の cursor で進む
ワークフローに組み込むエージェント不要

curl ひとつでどこへでも

サーバーは動いている間、自分で巡回します。変化をどこかへ流すには、HTTP ミラーをポーリングして転送します。カーソルは実行の合間にあなたが保持し、差分はサーバーが保持します。

crontab — 15 分ごと → Slack
# HN の新しい変化を Slack の webhook へ転送
*/15 * * * *  C=$(cat ~/.wfa_cursor 2>/dev/null); \
  R=$(curl -s "http://127.0.0.1:8848/changes?since=$C" -H "Authorization: Bearer $KEY"); \
  echo $R | jq '.cursor' > ~/.wfa_cursor; \
  echo $R | jq -c '.events[]' | while read e; do curl -s -X POST $SLACK_WEBHOOK -d "$e"; done

{"events": [], "cursor": N} を返す list_changes は、消音されたホルンです — 上流で何も変わっていない。これはエラーではなく、正常な状態です。

現場メモ

つまずきやすいところ

症状何が起きているか対処
403 Forbidden がトップページで出るサイトがブラウザの User-Agent なしのリクエストを拒否している。付ける:headers:{"User-Agent":"Mozilla/5.0 …"}
200 OK なのに「no feed found」検出できる <link rel=alternate> がなく、よくあるパスも解決しなかった。フィードの URL を直接渡す(多くは /feed か /rss)。
エージェントがツールを見つけられないMCP サーバーをセッション開始後に追加した。サーバー登録後にエージェントのセッションを再起動する。
すべての呼び出しが unauthorized接続に Authorization ヘッダーが付いていない — 複数行コマンドの貼り付けでは --header 行が落ちがち。ヘッダー付きで登録し直す(copy ボタンを使えば 1 行)。
2 回目のポーリングで同じ項目が出るlist_changes はカーソルを代わりに覚えてくれない。返ってきた cursor を since_cursor として渡す。
もっと多くの項目が欲しい件数の調整つまみはない — 項目数はフィードが出す通り(HN ≈ 30、多くのブログ ≈ 10–15)。ソース自身のページ送りフィード URL があれば、それを使う。
