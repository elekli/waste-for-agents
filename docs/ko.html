{ "events": [{ "kind":"added",
    "title":"Show HN: …",
    "content":"<clean markdown>" }],
  "cursor": 42 }
5분이면 갖게 되는 것

잠든 사이에도 도는 폴러

로컬 서버가 일정에 따라 피드를 돌며 진짜 변화만 쌓아 둡니다. 전달은 두 가지:에이전트용 MCP 도구(Claude Code 등), 그리고 평범한 HTTP 엔드포인트 — GET /changes — 로, curl 하나로 cron, Slack 웹훅, 다른 서비스에 흘려보낼 수 있습니다.

진짜 diff — timestamp 잡음 무시 본문은 깔끔한 Markdown 침묵이 곧 no-op
i

사전 준비. Python 3.12+, uv, 그리고 MCP 지원 에이전트 — Claude Code, OpenCode, Cursor(에이전트 경로에만 필요; HTTP 경로는 curl 만 있으면 됩니다).

설치 · 셀프 호스트

첫 배달까지 네 단계

1

클론하고 서버 시작

--unmetered 는 과금 게이트를 끕니다 — 자기 머신에서 돌리는데 호스팅 서비스용 무료 할당량에 막힐 이유가 없습니다.

터미널
# uv 가 없다면?  curl -LsSf https://astral.sh/uv/install.sh | sh  — 그 후 셸을 다시 여세요
# 산출물은 ~/.waste-for-agents/ 에 저장된다
git clone https://github.com/elekli/waste-for-agents.git && cd waste-for-agents
uv run python -m waste_for_agents serve --port 8848 --unmetered &
curl http://127.0.0.1:8848/health   # → {"status":"ok","watches":0}
2

직접 키 발급

로컬 DB에 키를 바로 써 넣고 출력합니다. (에이전트가 직접 발급할 수도 있습니다 — issue_key 는 키 없이 동작하는 유일한 도구입니다 — 하지만 결국 그 키를 클라이언트 설정에 붙여 넣고 다시 연결해야 합니다. CLI 로 먼저 발급하면 그 왕복을 줄일 수 있습니다.)

터미널
KEY=$(uv run python -m waste_for_agents issue-key)
echo $KEY   # → wfa_…
3

MCP 서버 등록 에이전트 경로

!

순서가 중요. 에이전트 세션을 시작(또는 재시작)하기 전에 서버를 등록하세요. 세션 도중에 추가한 서버의 도구는 불러오지 않습니다. 그리고 Authorization 헤더를 꼭 유지하세요 — 없으면 issue_key 를 제외한 모든 호출이 unauthorized 를 반환합니다.

터미널
claude mcp add --transport http posthorn http://127.0.0.1:8848/mcp/ --header "Authorization: Bearer $KEY"
# 그런 다음 새 Claude Code 세션을 시작
opencode.json — 프로젝트 루트 또는 (~/.config/opencode/)
{
  "mcp": {
    "posthorn": {
      "type": "remote",
      "url": "http://127.0.0.1:8848/mcp/",
      "headers": { "Authorization": "Bearer wfa_PASTE-YOUR-KEY" },
      "enabled": true
    }
  }
}

JSON 은 $KEY 를 확장하지 않습니다 — 2단계의 키(echo $KEY)를 그대로 붙여 넣고 OpenCode 를 재시작하세요.

.cursor/mcp.json (~/.cursor/mcp.json)
{
  "mcpServers": {
    "posthorn": {
      "url": "http://127.0.0.1:8848/mcp/",
      "headers": { "Authorization": "Bearer wfa_PASTE-YOUR-KEY" }
    }
  }
}

2단계의 키를 그대로 붙여 넣고 Cursor Settings → MCP 에서 서버를 다시 로드(또는 Cursor 재시작)하세요.

4

구독하고 빼내기

에이전트에서: watch 를 만들고 변화를 끌어옵니다. 홈페이지 URL 을 주면 디스커버리가 피드를 찾습니다(<link>, 그다음 /feed, /rss … 순서로 시도).

에이전트에서
w = create_watch(
  source="rss",
  query={"url": "https://news.ycombinator.com/news"},
  key_columns=["id"], ignore_columns=[], interval_s=3600)   # → {"watch_id": ...}

list_changes(watch_id=w["watch_id"])   # 이 watch의 변경 — 각 watch는 자기 cursor로 소진
워크플로에 연결에이전트 불필요

curl 하나로 어디든

서버는 돌아가는 동안 알아서 폴링합니다. 변화를 어딘가로 보내려면 HTTP 미러를 폴링해 전달하세요. 커서는 실행 사이에 당신이, diff 는 서버가 보관합니다.

crontab — 15분마다 → Slack
# HN 의 새 변화를 Slack 웹훅으로 전달
*/15 * * * *  C=$(cat ~/.wfa_cursor 2>/dev/null); \
  R=$(curl -s "http://127.0.0.1:8848/changes?since=$C" -H "Authorization: Bearer $KEY"); \
  echo $R | jq '.cursor' > ~/.wfa_cursor; \
  echo $R | jq -c '.events[]' | while read e; do curl -s -X POST $SLACK_WEBHOOK -d "$e"; done

{"events": [], "cursor": N} 을 돌려주는 list_changes 는 음소거된 나팔입니다 — 상류에서 바뀐 게 없다는 뜻. 오류가 아니라 정상 상태입니다.

현장 메모

사람들이 자주 걸리는 곳

증상무슨 일인가해결
403 Forbidden 이 홈페이지에서사이트가 브라우저 User-Agent 없는 요청을 막는다.하나 추가: headers:{"User-Agent":"Mozilla/5.0 …"}
200 OK 인데 "no feed found"찾을 수 있는 <link rel=alternate> 가 없고, 흔한 경로도 해결되지 않음.피드 URL 을 직접 지정(보통 /feed 또는 /rss).
에이전트가 도구를 못 본다세션이 시작된 뒤에 MCP 서버를 추가했다.서버 등록 후 에이전트 세션을 재시작.
모든 호출이 unauthorized연결에 Authorization 헤더가 없습니다 — 여러 줄 명령을 붙여 넣을 때 --header 줄이 빠지기 쉽습니다.헤더를 붙여 다시 등록하세요(copy 버튼을 쓰면 한 줄입니다).
두 번째 폴링에서 같은 항목 반복list_changes 는 커서를 대신 기억해 주지 않는다.돌려받은 cursor 를 since_cursor 로 넘긴다.
받은 것보다 더 많은 항목이 필요개수 조절 손잡이가 없음 — 항목 수는 피드가 내보내는 대로(HN ≈ 30, 많은 블로그 ≈ 10–15).소스 자체의 페이지네이션 피드 URL 이 있으면 그걸 사용.
