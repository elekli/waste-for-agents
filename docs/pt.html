Um coletor que roda enquanto você dorme

Um servidor local consulta seus feeds em intervalos e acumula apenas as mudanças reais. Ele as entrega de duas formas: como ferramentas MCP para o seu agente (Claude Code e afins) e como um endpoint HTTP simples — GET /changes — que você encaminha com curl para um cron, um webhook do Slack ou outro serviço.

Pré-requisitos. Python 3.12+, uv e um agente com suporte a MCP — Claude Code, OpenCode ou Cursor (apenas para o caminho do agente; o caminho HTTP precisa só de curl).

Quatro passos até a sua primeira entrega

1

Clone e inicie o servidor

--unmetered desliga o portão de cobrança — você está na sua própria máquina; não deveria ser limitado por uma cota gratuita pensada para o serviço hospedado.

terminal
# ainda sem uv?  curl -LsSf https://astral.sh/uv/install.sh | sh  — depois reabra o shell
# o estado fica em ~/.waste-for-agents/
git clone https://github.com/elekli/waste-for-agents.git && cd waste-for-agents
uv run python -m waste_for_agents serve --port 8848 --unmetered &
curl http://127.0.0.1:8848/health   # → {"status":"ok","watches":0}
2

Emita uma chave para você

Grava uma chave direto no banco local e a imprime. (Seu agente poderia emitir uma sozinho — issue_key é a única ferramenta que funciona sem chave — mas você ainda teria que colar a chave na configuração do cliente e reconectar. Emitir pela CLI primeiro poupa essa ida e volta.)

terminal
KEY=$(uv run python -m waste_for_agents issue-key)
echo $KEY   # → wfa_…
3

Registre o servidor MCP caminho do agente

!

A ordem importa. Registre o servidor antes de iniciar (ou reiniciar) a sessão do seu agente — um servidor adicionado no meio da sessão não carrega suas ferramentas. E mantenha o cabeçalho Authorization: sem ele, toda chamada exceto issue_key retorna unauthorized.

terminal
claude mcp add --transport http waste http://127.0.0.1:8848/mcp/ --header "Authorization: Bearer $KEY"
# depois inicie uma nova sessão do Claude Code
opencode.json — raiz do projeto ou (~/.config/opencode/)
{
  "mcp": {
    "waste": {
      "type": "remote",
      "url": "http://127.0.0.1:8848/mcp/",
      "headers": { "Authorization": "Bearer wfa_PASTE-YOUR-KEY" },
      "enabled": true
    }
  }
}

JSON não expande $KEY — cole a chave literal do passo 2 (echo $KEY) e reinicie o OpenCode.

.cursor/mcp.json (~/.cursor/mcp.json)
{
  "mcpServers": {
    "waste": {
      "url": "http://127.0.0.1:8848/mcp/",
      "headers": { "Authorization": "Bearer wfa_PASTE-YOUR-KEY" }
    }
  }
}

Cole a chave literal do passo 2 e recarregue o servidor em Cursor Settings → MCP (ou reinicie o Cursor).

4

Assine e drene

No seu agente: crie um watch e puxe as mudanças. Passe a URL da página inicial — a descoberta encontra o feed (tenta <link>, depois /feed, /rss…).

no seu agente

list_changes()   # um lote no primeiro despertar, depois silêncio até algo mudar
Conecte a um fluxo de trabalhosem agente

Um curl para qualquer coisa

O servidor consulta sozinho enquanto roda. Para enviar as mudanças a qualquer lugar, consulte o espelho HTTP e encaminhe. Você guarda o cursor entre as execuções; o servidor guarda o diff.

crontab — a cada 15 min → Slack
# encaminha novas mudanças do HN para um webhook do Slack
*/15 * * * *  C=$(cat ~/.wfa_cursor 2>/dev/null); \
  R=$(curl -s "http://127.0.0.1:8848/changes?since=$C" -H "Authorization: Bearer $KEY"); \
  echo $R | jq '.cursor' > ~/.wfa_cursor; \
  echo $R | jq -c '.events[]' | while read e; do curl -s -X POST $SLACK_WEBHOOK -d "$e"; done

Um list_changes que retorna {"events": [], "cursor": N} é a corneta silenciada — nada mudou na origem. Esse é o estado normal, não um erro.
